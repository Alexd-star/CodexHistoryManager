# Codex 本地对话历史管理器设计说明

## 一、目标

把临时的聊天恢复脚本产品化为一个可重复使用的软件应用，用于管理 Codex 本地对话历史。

应用应支持：

- 列表查看本地会话；
- 预览聊天内容；
- 归档和恢复会话；
- 批量归档、批量恢复和批量导出；
- 修复会话索引；
- 备份 Codex 本地状态；
- 查看备份记录和操作记录；
- 导出聊天内容、图片和文档合集。

## 二、架构

采用桌面应用优先、本地 Web 备用的结构：

- 核心：`app.py` 负责会话读取、搜索、导出、备份、归档和索引修复；
- 桌面端：`modern_app.py` 使用 CustomTkinter 构建现代化 Windows 桌面界面；
- 备用 Web 端：`static/index.html` + Python 标准库 HTTP 服务；
- 默认数据源：`C:\Users\<用户>\.codex`，可在界面中选择其他目录；
- 用户数据目录：`%LOCALAPPDATA%\CodexHistoryManager`；
- 导出目录：用户数据目录下的 `exports`；
- 备份目录：用户数据目录下的 `backups`；
- 日志目录：用户数据目录下的 `logs`；
- 配置文件：用户数据目录下的 `config.json`。

桌面端依赖 `customtkinter` 与 `pillow`，打包使用 `pyinstaller`。备用 Web 端保留为本机浏览器查看和接口调试入口。

程序目录只作为资源目录使用，包含 EXE、图标、静态页面等只读资源。发布版 EXE 位于只读目录、`Program Files` 或网盘同步目录时，也不会把导出、备份、日志写到程序目录。

## 三、数据读取

会话来源包括：

1. `state_5.sqlite` 的 `threads` 表；
2. `session_index.jsonl` 的标题和更新时间；
3. `sessions` 目录下的 JSONL 文件；
4. `archived_sessions` 目录下的 JSONL 文件。

列表数据优先使用 SQLite，文件系统扫描作为补充，避免索引损坏时看不到会话。

## 四、写操作策略

归档、恢复、索引修复都属于写操作。

执行前必须自动备份：

- `state_5.sqlite`
- `logs_2.sqlite`
- `session_index.jsonl`
- `config.toml`
- `process_manager/chat_processes.json`
- 目标会话 JSONL

归档和恢复只更新 SQLite 中的 `archived` 与 `archived_at` 字段，不移动原始 JSONL 文件。

批量归档和批量恢复使用同一个事务更新多个会话，执行前生成一次包含所有目标会话的备份。这样既能减少重复备份，也方便事后追溯。

所有关键操作会追加写入用户日志目录下的 `操作日志.jsonl`，包括：

- 备份；
- 导出；
- 归档；
- 恢复。

操作日志只记录操作元数据，不复制聊天正文。

应用异常会写入用户日志目录下的 `应用日志.log`。桌面端管理页提供诊断中心，显示应用目录、资源目录、用户数据目录、配置文件、路径状态、目录可写性、会话数量、备份数量、最近操作和最近备份，方便用户复制给开发者排查问题。

## 五、导出策略

导出格式：

- Markdown；
- HTML；
- TXT；
- JSON。

导出模式：

- 当前会话；
- 当前筛选列表；
- 多会话合并；
- 多会话分文件。

图片处理：

- 从 `input_image` 的 data URL 中提取；
- 按 SHA256 摘要命名；
- 保存到导出包的 `images` 子目录；
- Markdown/HTML 使用相对路径引用图片。

## 六、错误处理

常见错误和处理：

- JSONL 损坏行：跳过该行，继续解析；
- SQLite 暂时锁定：使用超时等待；
- 会话文件缺失：列表中标记文件缺失；
- 首次启动没有会话数据：显示中文引导，提示用户选择正确的 `.codex` 目录；
- 配置文件损坏：忽略损坏配置，回退到 `CODEX_HOME` 或 `~/.codex`；
- 下载越权路径：直接拒绝；
- 写操作失败：保留备份目录，返回错误信息。

桌面端错误弹窗只展示简短摘要和下一步建议，完整堆栈写入应用日志，避免弹窗被大量技术文本淹没。

## 七、性能策略

会话预览默认只解析最近若干条消息，并优先从 JSONL 文件尾部读取，避免把大体积 JSONL 一次性读入内存。

桌面界面采用以下优化：

- 会话列表分批渲染；
- 普通筛选预计算搜索索引；
- 点选、勾选、全选、清空只刷新必要行；
- 快速切换会话时预览加载防抖；
- 旧预览请求结果会自动丢弃；
- 超长消息先限量再排版。

完整统计需要遍历全量消息，因此在界面中作为“统计全量消息”按钮按需触发。导出仍然会读取全量消息，保证导出内容完整。
