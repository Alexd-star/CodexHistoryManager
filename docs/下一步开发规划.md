# 下一步开发规划

## 一、目标

把 Codex History Manager 从“能用的本地工具”推进到“公开仓库可持续维护、可自动验证、可发布 EXE、用户更容易上手”的产品状态。

下一步开发重点不是继续堆功能，而是补齐公开软件项目必须有的工程能力：

1. 公开仓库可自检；
2. 自动化构建与发布；
3. 发布包安全边界；
4. 更好的用户操作引导；
5. 更清晰的错误诊断；
6. 后续功能扩展的架构边界。

## 二、外部资料依据

本规划参考以下官方资料和最佳实践：

- PyInstaller 官方用法文档：支持通过 `--onefile`、`--windowed`、`--icon`、`--add-data` 等参数构建 Windows GUI 程序。
- GitHub Releases 官方文档：Release 适合把可下载的软件版本和二进制资产提供给用户。
- GitHub CLI `gh release create` 官方手册：支持创建 release 并上传本地构建产物。
- GitHub Actions 官方文档：工作流可在 push、pull request、tag 等事件上自动运行。
- GitHub Secret Scanning 官方文档：公开仓库需要避免提交 token、密钥、Cookie、导出聊天记录和本地备份。

## 三、当前状态评估

### 已完成

- 桌面版主程序：`modern_app.py`
- 核心数据逻辑：`app.py`
- 备用 Web 界面：`static/index.html`
- 中文 README
- MIT License
- `.gitignore`
- 本地冒烟测试：`tests/冒烟测试.py`
- GitHub 公开仓库

### 主要缺口

1. `tests/冒烟测试.py` 依赖本机真实 `.codex`，GitHub Actions 上不可运行。
2. 没有 CI 工作流，公开仓库无法自动证明代码可运行。
3. 没有 Release 工作流，用户无法直接在 GitHub Releases 下载 EXE。
4. README 说明了打包方式，但缺少“下载发布版”的路径。
5. 代码集中在 `app.py` 和 `modern_app.py`，后续应逐步拆分，但不能一次性大重构。
6. 缺少版本号、变更日志和发布纪律。
7. 错误提示仍偏工程化，普通用户遇到 SQLite 锁定、路径错误、权限错误时需要更明确解释。

## 四、优先级路线图

### P0：公开仓库工程化

目标：任何人 clone 仓库后，都能通过一个不依赖真实 `.codex` 的自检测试确认核心功能可用。

任务：

- 新增 `tests/公开仓库自检.py`；
- 使用临时目录模拟 `.codex` 结构；
- 生成最小 `state_5.sqlite`、`session_index.jsonl`、`rollout-*.jsonl`；
- 覆盖列表读取、消息读取、搜索、导出、备份、索引修复；
- 新增 `.github/workflows/ci.yml`；
- CI 在 Windows 上安装依赖并运行公开自检；
- 保留本地真实数据冒烟测试，但不放入 CI。

验收：

- 本地运行 `python .\tests\公开仓库自检.py` 通过；
- GitHub Actions 能运行该测试；
- 不访问真实用户 `.codex`；
- 不生成会被 Git 提交的导出/备份内容。

### P1：自动构建与发布

目标：打 tag 后自动构建 Windows EXE，并发布到 GitHub Releases。

任务：

- 新增 `.github/workflows/release.yml`；
- 在 `v*` tag 上触发；
- 安装依赖；
- 使用 PyInstaller 构建 `CodexHistoryManager.exe`；
- 上传构建产物为 workflow artifact；
- 创建 GitHub Release；
- 把 EXE 作为 release asset 上传。

验收：

- 本地打包脚本仍可用；
- CI 打包命令不依赖本机路径；
- Release 中出现 EXE 下载项。

### P2：版本号与变更日志

目标：每个发布版本可追溯。

任务：

- 新增 `VERSION`；
- 新增 `CHANGELOG.md`；
- README 显示当前版本；
- 打包脚本打印版本；
- Release 工作流读取版本或 tag。

验收：

- 用户能知道当前版本；
- 发布说明有历史记录；
- 版本号和 tag 对齐。

### P3：用户体验与错误诊断

目标：普通用户遇到问题能知道下一步做什么。

任务：

- 错误提示分类：路径不存在、SQLite 锁定、JSONL 缺失、导出失败、备份失败；
- 增加“诊断信息”面板；
- 增加“打开备份目录”“打开导出目录”“打开日志目录”；
- 增加“复制诊断信息”按钮；
- README 增加常见问题。

验收：

- 常见错误不只显示 Python 异常；
- 用户能复制诊断信息；
- 不泄露聊天正文。

### P4：功能扩展

目标：在安全和稳定的基础上增强产品能力。

候选功能：

- 会话标签和备注；
- 收藏会话；
- 导出 DOCX；
- 导出 PDF；
- 图片缩略图预览；
- 导出前预估文件大小；
- 自动定时备份；
- 多 Codex 数据目录配置；
- 操作日志可视化；
- 恢复前差异预览。
