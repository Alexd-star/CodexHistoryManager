# 安装包与发布说明

## 目标

为普通客户提供比单文件 EXE 更完整的软件安装体验，包括开始菜单快捷方式、可选桌面快捷方式、卸载入口和 Release 自动发布资产。

## 发布资产

每次正式版本发布应包含两个文件：

1. `CodexHistoryManager.exe`

   绿色版单文件程序。适合临时试用、放到 U 盘或不想安装的用户。

2. `CodexHistoryManager-Setup-v版本号.exe`

   正式安装包。推荐普通客户使用。

## 安装策略

安装包使用 Inno Setup 6 构建，默认安装到：

```text
%LOCALAPPDATA%\Programs\CodexHistoryManager
```

这样做的原因：

- 不需要管理员权限；
- 适合学校、公司等权限受限电脑；
- 避免写入 `Program Files` 后遇到权限问题；
- 安装和卸载都只影响当前 Windows 用户。

安装器界面使用简体中文语言文件。该语言文件来自 `kira-96/Inno-Setup-Chinese-Simplified-Translation`，授权文件随安装包源码放在 `installer/languages/`，并会随应用安装到 `licenses` 子目录。

## 快捷方式

安装包会创建：

- 开始菜单：`Codex History Manager`
- 开始菜单：`打开用户数据目录`
- 开始菜单：`卸载 Codex History Manager`
- 桌面快捷方式：可选，默认不勾选

## 用户数据保留策略

卸载程序只删除应用本体，不删除用户数据目录：

```text
%LOCALAPPDATA%\CodexHistoryManager
```

该目录包含：

- 导出结果；
- 自动备份；
- 应用日志；
- 用户选择过的 Codex 数据目录配置。

保留这些数据更安全，避免客户卸载软件时误删重要聊天导出和备份。若客户确认不再需要，可手动删除该目录。

## 本地构建

先构建绿色版 EXE：

```powershell
powershell -ExecutionPolicy Bypass -File .\打包桌面版EXE.ps1
```

再构建安装包：

```powershell
powershell -ExecutionPolicy Bypass -File .\打包安装包.ps1
```

如果没有安装 Inno Setup 6，脚本会给出安装提示。

## GitHub Release

推送 `v*` 标签时，Release 工作流会：

1. 安装 Python 依赖；
2. 运行公开仓库自检；
3. 构建绿色版 EXE；
4. 安装 Inno Setup；
5. 构建安装包；
6. 静默安装到临时目录并静默卸载，验证安装包可用；
7. 上传 EXE 和安装包到 GitHub Release。

## 发布检查清单

发布前应确认：

- `VERSION` 与标签版本一致；
- `README.md` 当前版本已更新；
- `CHANGELOG.md` 已归档版本变更；
- 公开仓库自检通过；
- 语法解析检查通过；
- 本地 EXE 打包通过；
- 安装包打包通过；
- `tests/installer_smoke_test.ps1` 安装/卸载冒烟测试通过；
- Release 页面包含绿色版 EXE 和 Setup 安装包两个资产。
