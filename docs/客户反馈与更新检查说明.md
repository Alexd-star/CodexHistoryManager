# 客户反馈与更新检查说明

## 目标

提升客户支持效率。用户遇到问题时，可以在桌面端一键生成反馈包；需要升级时，可以直接检查 GitHub Releases 最新版本。

## 客户反馈包

入口：

```text
管理 > 产品支持 > 生成反馈包
```

反馈包默认保存到导出目录：

```text
%LOCALAPPDATA%\CodexHistoryManager\exports
```

反馈包内容：

- `诊断信息.json`
- `诊断信息.txt`
- `操作记录.json`
- `应用日志尾部.log`
- `操作日志尾部.jsonl`
- `说明.txt`

反馈包不包含：

- Codex 会话正文；
- `sessions` 目录下的原始 JSONL；
- `archived_sessions` 目录下的原始 JSONL；
- 用户导出的 Markdown、HTML、TXT、JSON 聊天内容。

注意：反馈包可能包含本机路径、用户名、版本号、目录状态和最近操作元数据。发送给他人前，应确认这些路径信息可以公开。

## 更新检查

入口：

```text
管理 > 产品支持 > 检查更新
```

程序会访问：

```text
https://api.github.com/repos/Alexd-star/CodexHistoryManager/releases/latest
```

检查逻辑：

1. 读取本地 `VERSION`；
2. 请求 GitHub Releases 最新版本；
3. 比较语义版本号；
4. 若发现新版本，提示用户打开下载页面。

如果网络不可用、GitHub API 限流或 VPN/代理异常，程序不会影响本地功能，只提示用户手动打开发布页：

```text
https://github.com/Alexd-star/CodexHistoryManager/releases/latest
```

## 设计边界

- 更新检查只提示，不自动下载、不自动安装。
- 反馈包只用于诊断，不作为聊天内容导出功能。
- 反馈包尽量收集足够排查的信息，同时避免包含聊天正文。
