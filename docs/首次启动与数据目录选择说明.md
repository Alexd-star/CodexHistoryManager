# 首次启动与数据目录选择说明

## 适用场景

首次打开 Codex History Manager 后，如果会话列表为空，通常表示当前读取的数据目录中没有发现可识别的 Codex 历史文件。该情况不等同于历史记录丢失，也不代表软件无法使用。

## 正确的数据目录

软件需要读取 Codex 本地数据目录。一个有效目录通常包含以下任意一种文件或文件夹：

- `state_5.sqlite`
- `session_index.jsonl`
- `sessions`

Windows 用户常见位置为：

```text
%USERPROFILE%\.codex
```

如果用户曾经手动迁移过 Codex 数据，或在不同账号、不同工作区中使用过 Codex，应选择实际保存这些文件的 `.codex` 目录。

## 首次启动操作流程

1. 启动 `CodexHistoryManager.exe`。
2. 如果列表为空，查看界面右侧“连接你的 Codex 本地历史”区域。
3. 点击“选择数据目录”。
4. 选择包含 `state_5.sqlite`、`session_index.jsonl` 或 `sessions` 的 `.codex` 目录。
5. 软件会自动刷新会话列表，并记住本次选择。
6. 后续启动时会继续使用已保存的数据目录。

## 如何判断目录选择正确

选择正确目录后，软件应显示会话列表。每个会话通常包含标题、更新时间、文件大小、状态标签和预览信息。

如果仍然为空，请检查：

- 本机是否已经使用 Codex 产生过至少一次对话。
- 选择的目录是否只是 `.codex` 的父目录，而不是 `.codex` 本身。
- 目录中是否存在 `sessions` 文件夹或 `state_5.sqlite` 文件。
- 当前 Windows 用户是否有读取该目录的权限。

## 诊断与反馈

如果按上述步骤仍无法显示会话，请在软件中执行：

1. 打开“管理”页。
2. 找到“诊断中心”。
3. 点击“刷新诊断”。
4. 点击“复制诊断”或“生成客户反馈包”。

客户反馈包不会包含聊天正文和原始会话 JSONL 内容。它只包含诊断信息、日志尾部和必要的运行环境信息，便于维护人员定位路径、权限或配置问题。

## 数据安全说明

Codex History Manager 默认只读取本机 `.codex` 历史数据。导出、备份、日志和配置会写入：

```text
%LOCALAPPDATA%\CodexHistoryManager
```

软件不会主动上传聊天内容。用户只有在手动导出、备份或生成反馈包时，才会产生对应文件。

## 常见问题

### 为什么软件没有自动找到我的历史记录？

不同 Codex 版本、不同启动方式或不同 Windows 用户账号可能使用不同的数据目录。软件会优先读取已保存配置，其次尝试默认 `.codex` 目录。若实际目录不在默认位置，需要手动选择一次。

### 选择目录后为什么下次还能记住？

软件会把用户选择的数据目录保存到：

```text
%LOCALAPPDATA%\CodexHistoryManager\config.json
```

该文件只保存路径等配置，不保存聊天正文。

### 可以选择任意文件夹吗？

可以选择，但只有包含 Codex 历史文件结构的目录才能显示会话。推荐直接选择 `.codex` 目录，而不是上级目录。
